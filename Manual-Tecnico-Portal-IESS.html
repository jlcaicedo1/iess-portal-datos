1. Introducción

Propósito y alcance del manual

Este manual técnico describe cómo administrar, actualizar y escalar el Portal Maestro Institucional de Gestión de Tratamiento de Datos Personales del IESS, archivo Portal-IESS-DataProtection.html.

ℹ️
Formato standalone El portal es un único archivo HTML independiente. No requiere servidor web, frameworks ni conexión a internet para funcionar en red local. Todo el CSS, JavaScript y el logo institucional están embebidos dentro del mismo archivo.

Herramientas necesarias para editar

  • Editor de texto recomendado: Visual Studio Code (gratuito) — code.visualstudio.com
  • Alternativa: Notepad++, Sublime Text
  • Navegador para previsualizar: Google Chrome, Microsoft Edge
  • Para conversión de logo a base64: Python 3 (ya instalado en el sistema)

Convenciones usadas en este manual

SímboloSignificado
código en grisFragmento de código HTML que debe buscarse o editarse en el archivo
GSI-P05-I01Código del documento según nomenclatura institucional
DisponibleDocumento ya publicado y con hipervínculo activo
PendienteDocumento aún sin publicar — hipervínculo deshabilitado

2. Estructura del Portal

Árbol de archivos y secciones lógicas

2.1 Árbol de archivos esperado

El portal asume la siguiente estructura de carpetas. Todos los archivos HTML deben estar en la misma carpeta; los PDF están un nivel arriba.

3. Cómo abrir el portal

Uso local y en red

Apertura local (PC individual)

Hacer doble clic sobre Portal-IESS-DataProtection.html. El navegador predeterminado lo abrirá directamente. No se necesita internet ni servidor.

Apertura en red local (LAN institucional)

1
Compartir la carpeta HTML
En el equipo que actuará como servidor: clic derecho sobre la carpeta HTML/ → Propiedades → Compartir → dar acceso de lectura a los usuarios de la red.
2
Acceder desde otro equipo
Desde el equipo cliente, en el Explorador de Windows escribir: \\NOMBRE-SERVIDOR\HTML\Portal-IESS-DataProtection.html y presionar Enter. Se abrirá en el navegador.
3
Crear acceso directo
Arrastrar el archivo desde la ruta de red al escritorio del usuario para crear un acceso directo permanente.
✅
Ventaja standalone Como el logo y todo el contenido están embebidos, el portal se ve correctamente aunque el usuario no tenga acceso a internet o a servidores externos.

4. Agregar documentos

Cómo activar y enlazar documentos en el portal

4.1 Enlazar un archivo HTML nuevo

Cuando un documento HTML esté listo (por ejemplo, el Manual principal GSI-P05-Manual.html):

1
Copiar el archivo HTML a la carpeta
Colocar el archivo en Desktop\Sistema DataProtection\HTML\ — la misma carpeta donde está el portal.
2
Abrir el portal en VS Code
Clic derecho sobre Portal-IESS-DataProtection.html → "Abrir con" → Visual Studio Code.
3
Buscar la tarjeta del documento
Presionar Ctrl+F y buscar el código del documento, por ejemplo GSI-P05 o el título del documento que quieres activar.
4
Cambiar el estado de "pending" a "available"
Cada tarjeta de documento pendiente tiene esta estructura. Realiza los 3 cambios marcados:

Estructura actual (pendiente):

HTML — antes <article class="doc-card pending" ...> ... <div class="doc-card-footer"> <div class="doc-status pending" ...> <span class="status-dot pending"></span> Pendiente publicación </div> <a href="#" class="doc-link disabled" aria-disabled="true" tabindex="-1"> Próximamente </a> </div> </article>

Estructura activada (disponible):

HTML — después <article class="doc-card available" ...> ... <div class="doc-card-footer"> <div class="doc-status available" ...> <span class="status-dot available"></span> Disponible </div> <a href="GSI-P05-Manual.html" class="doc-link active" target="_blank" rel="noopener"> Abrir documento </a> </div> </article>
💡
Resumen de los 3 cambios (1) pending → available en la clase del article y en el doc-status/status-dot. (2) href="#" → el nombre del archivo HTML. (3) disabled → active en la clase del enlace, y quitar aria-disabled y tabindex="-1".

4.2 Enlazar un archivo PDF

Si el documento es un PDF, el proceso es idéntico pero la ruta cambia según dónde esté el archivo:

Ubicación del PDFRuta a usar en href
Carpeta HTML/ (misma que el portal)MiDocumento.pdf
Carpeta Sistema DataProtection/ (un nivel arriba)../MiDocumento.pdf
Subcarpeta dentro de HTML/subcarpeta/MiDocumento.pdf

4.3 Activar un Instructivo (dentro del Manual)

Los instructivos están en el tab "Instructivos" del accordion del Manual. Buscar en el portal (Ctrl+F) el código del instructivo, por ejemplo GSI-P05-I01, y aplicar los mismos 3 cambios del punto 4.1 sobre las clases sub-link:

Antes (pendiente) <a href="#" class="sub-link disabled" aria-disabled="true" tabindex="-1"> Ver documento </a>
Después (activo) <a href="GSI-P05-I01-Legitimidad.html" class="sub-link active" target="_blank" rel="noopener"> Ver instructivo </a>

También cambiar el bloque doc-status de la tarjeta de pending a available:

<div class="doc-status available" style="font-size:10.5px"> <span class="status-dot available" aria-hidden="true"></span>Disponible </div>

4.4 Activar un Formato

Los formatos están en el tab "Formatos". El proceso es idéntico al instructivo. Buscar GSI-P05-F01 (o el número que corresponda) y aplicar los mismos cambios.

✅
Consejo Usa VS Code con la extensión "Live Preview" para ver los cambios en tiempo real mientras editas el portal. Instala la extensión buscando "Live Preview" en el panel de extensiones (Ctrl+Shift+X).

4.5 Agregar una nueva herramienta técnica

Para agregar una nueva tarjeta en la sección "Herramientas Técnicas", copiar el siguiente bloque dentro de <div class="tools-grid">:

Plantilla de tarjeta de herramienta <article class="tool-card" aria-labelledby="tool-NUEVO-title"> <div class="tool-card-stripe CLASE-COLOR" aria-hidden="true"></div> <div class="tool-card-body"> <div class="tool-icon CLASE-COLOR" aria-hidden="true"> <!-- SVG del ícono aquí --> </div> <h3 class="tool-title" id="tool-NUEVO-title">Nombre de la Herramienta</h3> <p class="tool-desc">Descripción de la herramienta...</p> <div class="tool-features"> <div class="tool-feature">Característica 1</div> <div class="tool-feature">Característica 2</div> </div> </div> <div class="tool-card-footer"> <a href="NombreArchivo.html" class="tool-btn primary" target="_blank" rel="noopener"> Abrir herramienta </a> </div> </article>

Las clases de color disponibles para tool-card-stripe y tool-icon son: eipdp (azul navy), check (verde), biblio (morado), ley (dorado). Para agregar un color nuevo, añadir en el CSS una regla como .tool-card-stripe.nuevo { background: #color; }.

5. Logo y branding institucional

Actualización del logo y paleta de colores

Cómo actualizar el logo

El logo del IESS está embebido como Base64 directamente en el HTML. Si el logo cambia, seguir estos pasos:

1
Obtener el nuevo archivo PNG del logo
Ruta de referencia del logo actual: E:\Gestión IESS\Imagen institucional\...\Logotipo-IESS-horizontal.png
2
Convertir a Base64 con Python
Abrir PowerShell y ejecutar:
PowerShell python -c " import base64 with open(r'RUTA\AL\LOGO.png', 'rb') as f: b64 = base64.b64encode(f.read()).decode() with open(r'logo_b64.txt', 'w') as out: out.write(b64) print('Listo:', len(b64), 'chars') "
3
Reemplazar en el portal
En el portal, buscar (Ctrl+F) data:image/png;base64,. Hay exactamente una ocurrencia. Seleccionar toda la cadena base64 (desde la coma hasta el cierre de comillas) y pegar el nuevo contenido del archivo logo_b64.txt.

6. Estados de documento

Sistema de estados y clases CSS

Cada tarjeta de documento tiene un estado visual que indica si el documento está disponible o pendiente:

EstadoClase articleClase statusClase enlaceVisual
Pendiente doc-card pending doc-status pending / status-dot pending doc-link disabled Borde izquierdo gris, enlace deshabilitado, texto "Pendiente"
Disponible doc-card available doc-status available / status-dot available doc-link active Borde izquierdo verde, enlace activo azul navy
Destacado doc-card featured — doc-link active Borde dorado, fondo ligeramente amarillo
ℹ️
Sub-cards (instructivos y formatos) Usan las clases sub-link active / sub-link disabled en lugar de doc-link, pero el principio es el mismo.

7. Tabs del Manual GSI-P05

Accordion y sistema de tabs

Cómo funciona el accordion

El Manual GSI-P05 tiene un botón que expande/colapsa el contenido. El JavaScript que lo controla es la función toggleManual(this). No es necesario modificarla.

Tabs internos del manual

Dentro del accordion hay 3 tabs: Instructivos, Formatos y Matrices y Diagrama. Cada tab corresponde a un <div class="manual-tab-content"> con su ID:

TabID del panelContenido
Instructivostab-instructivos7 sub-cards (I01 a I07)
Formatostab-formatos11 sub-cards (F01 a F11)
Matrices y Diagramatab-matrices3 sub-cards (RAT, EIPD, Diagrama)

Agregar un nuevo tab (si se necesita)

1
Agregar botón del tab
Dentro de <div class="manual-tabs">, copiar un botón existente y cambiar el texto, el aria-controls y el id del botón.
2
Agregar panel del tab
Debajo del último <div class="manual-tab-content">, agregar un nuevo div con el ID que corresponde al aria-controls del botón.

8. Migración a Google Drive / Workspace

Opciones para acceso externo sin servidor propio
⚠️
Limitación importante Google Drive no puede ejecutar archivos HTML interactivos directamente como sitio web (la función "Hospedar como sitio web" fue descontinuada). Sin embargo, existen opciones que funcionan muy bien para el caso del portal.

Opción A — Google Drive (solo descarga/apertura local)

Recomendado para uso interno básico. Sube todos los archivos a una carpeta compartida de Google Drive. El usuario descarga el portal y lo abre localmente.

1
Subir toda la carpeta HTML/ a Google Drive
En Google Drive, clic en "Nuevo" → "Subir carpeta" → seleccionar la carpeta HTML/ completa. También subir los PDF de la carpeta padre.
2
Compartir la carpeta
Clic derecho sobre la carpeta → "Compartir" → ingresar los correos institucionales o generar un enlace con "Cualquier usuario de la organización puede ver".
3
El usuario descarga y abre
El destinatario descarga el archivo Portal-IESS-DataProtection.html y lo abre en su navegador. Como el logo y CSS están embebidos, funciona perfectamente sin conexión.

Cambios necesarios para que los enlaces funcionen desde Drive

Cuando el portal se usa desde Google Drive descargado, los enlaces a los demás archivos HTML deben apuntar a archivos también descargados en la misma carpeta. Esto funciona sin cambios si el usuario descarga toda la carpeta completa.

Para los PDF, cambiar las rutas en el portal de ../archivo.pdf a solo archivo.pdf si se reubican los PDF dentro de la misma carpeta HTML/.

Opción B — Google Sites (portal web real sin costo adicional)

Recomendado para acceso web permanente. Google Sites permite crear un mini-sitio web institucional con URLs permanentes, usando tu cuenta de Google Workspace (@iess.gob.ec o equivalente).

1
Crear un Google Site
Ir a sites.google.com con la cuenta institucional → "Nuevo sitio" → elegir plantilla en blanco.
2
Insertar el portal como HTML embebido
En el editor de Sites: insertar → "Insertar" → "Insertar HTML" → pegar el contenido completo del portal. Limitación: Google Sites tiene restricciones de seguridad que pueden bloquear algunos estilos o scripts. Funciona mejor para páginas informativas con hipervínculos.
3
Subir los archivos HTML a Google Drive y obtener vínculos
Para que los hipervínculos funcionen desde Sites, cada archivo HTML (.html) debe subirse a Drive y se debe obtener su enlace de "Vista previa" para usarlo como href en el portal. Esto requiere actualización manual de cada enlace.

Opción C — GitHub Pages (recomendada para acceso web completo)

La solución más robusta y gratuita para publicar el portal exactamente como fue diseñado, con todas las funcionalidades activas.

1
Crear cuenta en GitHub
Registrar en github.com con el correo institucional. Crear un repositorio privado o público llamado iess-portal-datos.
2
Subir los archivos
Subir todos los archivos HTML y PDF al repositorio. El archivo principal debe llamarse index.html (renombrar el portal) o configurar el archivo principal en las opciones de Pages.
3
Habilitar GitHub Pages
En el repositorio: Settings → Pages → Source: "Deploy from branch" → branch: main → folder: / (root) → Save. En 2-3 minutos el portal estará en https://usuario.github.io/iess-portal-datos/

Ajuste de rutas para uso en web (GitHub Pages o Sites)

Si el portal se publica en web, los PDF no pueden estar como archivos locales. Las opciones son:

  • Subir los PDF a Google Drive y compartirlos como "Cualquiera con el enlace puede ver". Luego reemplazar los href="../archivo.pdf" por la URL de Drive (formato: https://drive.google.com/file/d/ID/view).
  • Subir los PDF al mismo repositorio de GitHub y referenciarlos como rutas relativas normales.

9. Mantenimiento del portal

Actualizaciones periódicas y control de versiones

Control de versiones recomendado

Antes de hacer cambios al portal, guardar una copia de seguridad con la fecha:

Portal-IESS-DataProtection.html ← versión activa Portal-IESS-DataProtection-2026-07-05.html ← copia de seguridad

Checklist de mantenimiento mensual

  • Verificar si hay nuevos documentos aprobados y activar sus tarjetas
  • Actualizar la fecha "Última actualización" en el subheader del portal
  • Verificar que los enlaces a PDF existentes siguen funcionando
  • Revisar que los archivos HTML referenciados no han sido renombrados
  • Si hubo cambios al logo institucional, actualizar el base64

Actualizar la fecha en el subheader

Buscar en el portal (Ctrl+F): Vigente · Julio 2026 y actualizar al mes y año correspondiente.

Actualizar la versión en el pie de página

Buscar: Portal v1.0 y el texto v1.0 · Julio 2026 en el footer. Actualizar según corresponda.
